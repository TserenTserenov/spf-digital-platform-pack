# ИИ-агенты платформы

## 1. Определение

**ИИ-агент** — системно-ориентированный компонент, выполняющий автономные функции с использованием LLM (ai_role: core).

## 2. Отличие от ассистентов

| Аспект | ИИ-ассистент | ИИ-агент |
|--------|--------------|----------|
| **Ориентация** | Человек | Система |
| **Интерфейс** | Диалоговый | Программный (MCP, API) |
| **Инициатива** | По запросу | Автономно |
| **Примеры** | Проводник | RouteGuide, ProgressAnalyst |

## 5. Ключевые агенты (детализация)

### 5.1. RouteGuide (AGT.002)

**Миссия:** Превращать размытые намерения в конкретный план саморазвития.

**Входы:**
- Данные Twin: профиль, ступень, показатели, история
- Методика ступеней Ученика
- Каталоги LMS и проектов

**Выходы:**
- `PTH.Student.PersonalRoute` — персональный маршрут
- `STGA.StudentStageAssigned` — оценка ступени
- Команды агентам TrajectoryPlanner, DailyAssignmentBuilder

**Метрики:**
- stage_accuracy >90%
- route_acceptance_rate >80%
- transition_success_rate >70%

### 5.2. TrajectoryPlanner (AGT.003)

**Миссия:** Детализировать маршрут в конкретную траекторию с учётом зависимостей.

**Алгоритм:**
1. Извлечь целевые навыки из маршрута
2. Найти skill gap
3. Построить граф зависимостей
4. Топологическая сортировка
5. Подобрать ресурсы
6. Распределить по времени

**Метрики:**
- plan_optimality >85%
- dependency_violations = 0
- time_accuracy ±20%

### 5.3. ProgressAnalyst (AGT.005)

**Миссия:** Превращать данные об активности в понятные инсайты.

**Функции:**
- Еженедельный/ежемесячный анализ
- Выявление блокеров и паттернов
- Прогнозирование прогресса
- Подготовка данных для RouteGuide

**Типы блокеров:**
- `time_constraint` — временные ограничения
- `stuck_on_step` — застревание на шаге
- `engagement_drop` — снижение вовлечённости
- `burnout_risk` — риск выгорания
